📊 1. Setup Apps Script — LSEG

Langkah-langkah menghubungkan Google Sheet LSEG dengan dashboard:

1
Buka Google Sheet LSEG

Buka spreadsheet LSEG yang berisi data pengguna di Google Sheets.

2
Buka Apps Script Editor

Klik menu Extensions → Apps Script. Editor kode akan terbuka di tab baru.

3
Paste Kode Apps Script

Hapus semua kode bawaan di editor, lalu salin dan tempel kode berikut:

📄 File kode: apps-script/lseg-apps-script.js
Buka file tersebut di folder dashboard, salin seluruh isinya, dan paste di editor Apps Script.
4
Deploy sebagai Web App
  1. Klik tombol "Deploy" (kanan atas) → "New deployment"
  2. Klik ikon ⚙️ di sebelah "Select type" → pilih "Web app"
  3. Isi konfigurasi:
    • Description: LSEG API
    • Execute as: Me (akun Anda)
    • Who has access: Anyone
  4. Klik "Deploy"
  5. Klik "Authorize access" jika diminta, lalu pilih akun Google Anda
5
Copy URL Deployment

Setelah deploy berhasil, akan muncul URL seperti:

https://script.google.com/macros/s/AKfycb.../exec

Salin URL tersebut.

6
Paste URL di Dashboard

Buka halaman LSEG di dashboard, paste URL pada kolom "Konfigurasi Sumber Data", lalu klik "Hubungkan".

⚠️ Keamanan (Token Lock): Kode Apps Script kini dilengkapi token pengunci keamanan (pknstanlib_secret_key_2026). Dashboard otomatis menyematkan token ini secara aman saat mengambil data, sehingga pihak luar tidak bisa mengakses data Google Sheet Anda tanpa token ini.
⚠️ Penting: Jika Anda mengupdate kode Apps Script, Anda perlu membuat deployment baru (bukan edit deployment lama) agar perubahan berlaku.

💻 2. Setup Apps Script — STATA

Langkahnya sama persis seperti LSEG, tetapi menggunakan file kode yang berbeda:

📄 File kode: apps-script/stata-apps-script.js
Buka Google Sheet STATA → Extensions → Apps Script → paste kode dari file di atas → Deploy sebagai Web App.

Setelah mendapatkan URL deployment, paste di halaman STATA pada dashboard.

📚 3. Upload Data E-Journal

Data E-Journal menggunakan format COUNTER report dari Emerald Insight.

1
Siapkan File COUNTER Report

File harus berformat .xlsx atau .csv dengan format COUNTER 5 (TR_J1). File yang diberikan oleh Emerald Insight biasanya sudah sesuai format ini.

2
Buka Halaman E-Journal

Navigasi ke halaman E-Journal melalui sidebar.

3
Upload File

Drag & drop file ke area upload, atau klik "Pilih File" untuk memilih file dari komputer Anda.

4
Data Otomatis Tampil

Setelah file diproses, grafik dan tabel akan otomatis muncul. Data disimpan di browser (localStorage) sehingga tetap tersedia saat Anda membuka dashboard kembali.

💡 Tips: Untuk memperbarui data, cukup upload file baru. Data lama akan otomatis digantikan.

🖥️ 4. Cara Menggunakan Dashboard

Navigasi

  • Gunakan sidebar di kiri untuk berpindah antar halaman
  • Pada perangkat mobile, klik ikon menu (☰) di kiri atas
  • Halaman Home menampilkan ringkasan semua e-resources

Filter Data

  • Filter Tahun: Tampilkan data untuk tahun tertentu
  • Filter Prodi: Tampilkan data untuk program studi tertentu (LSEG & STATA)
  • Filter Kategori: Tampilkan data Dosen, Mahasiswa, atau Lainnya
  • Pencarian: Cari jurnal berdasarkan judul (E-Journal)

Membaca Grafik

  • Bar chart: Menunjukkan perbandingan kuantitas antar kategori
  • Line chart: Menunjukkan tren dari waktu ke waktu
  • Doughnut chart: Menunjukkan proporsi/distribusi
  • Arahkan kursor ke grafik untuk melihat detail (tooltip)

Tabel Data

  • Tabel menampilkan data detail dengan paginasi
  • Gunakan tombol navigasi di bawah tabel untuk berpindah halaman

🔄 5. Update Data Periodik

LSEG & STATA (Otomatis)

Data LSEG dan STATA berasal dari Google Form yang terhubung ke Google Sheets. Setiap ada pengisian form baru, data otomatis masuk ke spreadsheet. Dashboard akan mengambil data terbaru setiap kali dimuat (dengan cache 30 menit).

💡 Untuk memaksa refresh data, hapus cache browser atau buka dashboard di jendela Incognito.

E-Journal (Manual)

Data E-Journal perlu diupload manual secara periodik:

  1. Download COUNTER report terbaru dari Emerald Insight
  2. Buka halaman E-Journal di dashboard
  3. Upload file baru — data lama akan otomatis digantikan

🔧 6. Troubleshooting

Data LSEG/STATA tidak muncul

  • Pastikan URL Apps Script sudah benar (diawali https://script.google.com/)
  • Pastikan Apps Script sudah di-deploy sebagai Web App dengan akses "Anyone"
  • Coba buka URL Apps Script langsung di browser — harus menampilkan data JSON
  • Jika ada perubahan kode, buat deployment baru

File E-Journal gagal diproses

  • Pastikan file berformat .xlsx atau .csv
  • File harus memiliki header "Title" di kolom A (format COUNTER report)
  • Coba buka file di Excel/Google Sheets terlebih dahulu untuk memastikan data terbaca

Grafik tidak tampil

  • Pastikan koneksi internet stabil (Chart.js dimuat dari CDN)
  • Coba refresh halaman (Ctrl+F5)
  • Buka Console browser (F12 > Console) untuk melihat error

Hosting di GitHub Pages

  1. Buat repository baru di GitHub
  2. Upload seluruh folder dashboard/ ke repository
  3. Buka Settings → Pages → Source: Deploy from branch → Branch: main, folder: / (root)
  4. Website akan tersedia di https://username.github.io/repo-name/
